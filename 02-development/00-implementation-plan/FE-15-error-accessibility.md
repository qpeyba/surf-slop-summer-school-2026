# FE-15. Обработка ошибок, edge-кейсы, accessibility

> Оценка: 1 день

## Задачи

### Обработка ошибок
- [ ] Валидация телефона (E.164 формат, `^\+[1-9][0-9]{1,14}$`) на клиенте в `PhoneMask.kt`
- [ ] Маппинг всех кодов ошибок API в человекочитаемые сообщения на русском:
  - `bad_request` → «Некорректный запрос»
  - `unauthorized` → «Требуется авторизация» (обрабатывается `UnauthorizedInterceptor`)
  - `forbidden` → «Доступ запрещён»
  - `not_found` → «Не найдено»
  - `slot_full` → «Мест больше нет»
  - `slot_cancelled` → «Класс отменён»
  - `slot_started` → «Класс уже начался»
  - `double_booking` → «Вы уже записаны на этот класс»
  - `already_cancelled` → «Бронь уже отменена»
  - `booking_not_active` → «Бронь не активна»
  - `booking_not_completed` → «Оценить можно только завершённый класс»
  - `invalid_rating` → «Оценка должна быть от 1 до 5»
  - `invalid_code` → «Неверный код, попробуйте снова»
  - `too_many_requests` → «Слишком много запросов. Подождите»
  - `idempotency_conflict` → «Конфликт идемпотентности»
  - `internal_error` → «Внутренняя ошибка сервера»
- [ ] Сетевая ошибка (no internet, timeout) → `ErrorState` с текстом «Нет соединения с сервером» + Retry

### Edge-кейсы
- [ ] Телефон без интернета: все экраны показывают ErrorState с Retry
- [ ] Быстрое переключение табов: не вызывает повторных запросов (debounce/launchWhenResumed)
- [ ] Поворот экрана: ViewModel переживает конфигурацию, состояние сохраняется
- [ ] Process death: после восстановления — перезагрузка данных
- [ ] Сворачивание/разворачивание приложения: не дублирует запросы
- [ ] Медленный ответ API: лоадеры не зависают, есть таймаут OkHttp (30s)
- [ ] Booking status «КлиентНеПришёл»: отображается в группе «Отменённые» с бейджем «Не пришёл»
- [ ] Пустые `photoUrls[]`: плейсхолдер в `PhotoCarousel` / `ClassCard`

### Accessibility
- [ ] Content description на всех иконках и изображениях (`contentDescription` в `Icon`, `Image`, `IconButton`)
- [ ] Минимальные touch targets 48×48dp (проверка через Accessibility Scanner)
- [ ] `StarRating`: текстовая метка «{n} из 5» доступна для TalkBack (через `semantics.contentDescription`)
- [ ] Disabled-кнопки: `contentDescription` «Недоступно» или причина (например «Нет мест»)
- [ ] Контрастность текста: 4.5:1 (обычный), 3:1 (крупный) — проверка через Accessibility Scanner
- [ ] Ошибки валидации под полями ввода: текст ошибки (не только красная обводка)
- [ ] `DateStrip`: выбранная дата имеет `contentDescription` «Выбрано: 15 июля»

## Критерий приёмки

- Все API-ошибки отображаются с понятным текстом на русском языке, а не с machine-readable code
- Accessibility Scanner (Google Play) не выявляет критических проблем на всех экранах
- TalkBack корректно озвучивает все элементы навигации, карточки, кнопки, звёзды
- При отключении интернета: ErrorState с Retry (не краш)
- Поворот экрана: состояние сохраняется, UI не крашится
- Process death: приложение восстанавливается в том же состоянии
