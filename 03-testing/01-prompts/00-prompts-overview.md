# Промпты для тестирования «Шеф-стол»

Согласно лекции Влада, тестирование с AI разбивается на три столпа:
1. **Ревью требований** — поиск противоречий, пробелов, вопросов к ТЗ
2. **Генерация тестов** — сначала план (ширина), потом детализация (глубина)
3. **Автотесты** — генерация кода тестов с последующим ревью и прогоном

Каждый этап использует отдельный промпт. Файлы промптов хранятся в гите, итеративно улучшаются.

## Структура

```
01-prompts/
├── 00-prompts-overview.md    ← этот файл
├── 01-review-prompt.md        ← промпт для ревью ТЗ
├── 02-test-plan-prompt.md     ← промпт для плана тестов (ширина)
├── 03-test-cases-prompt.md    ← промпт для генерации тест-кейсов (глубина, JSON)
├── 04-layout-tests-prompt.md  ← промпт для тестов вёрстки (layout)
├── 05-review-tests-prompt.md  ← промпт для ревью сгенерированных тестов
├── 06-auto-tests-prompt.md    ← промпт для генерации автотестов
└── 07-test-schema.json        ← JSON-схема тест-кейса для структурированной генерации
```

## Общие принципы (из лекции)

1. **Роль AI — генератор, человек — проверяющий.** AI пишет, мы ревьюим.
2. **Сверху-вниз.** Сначала план → ревью → детализация → ревью.
3. **Работаем с файлами**, не с чатом. Результат всегда в `.md`/`.json`.
4. **Разные агенты для разных задач.** Один генерит, другой ревьюит.
5. **Гигиена контекста.** Новый чат для каждого большого этапа.
6. **Не ваншотить.** С первого раза идеально не будет. Это нормально.
7. **Использовать мощные модели** для задач, где нужно думать (планирование, ревью).
8. **Структурные форматы** (JSON) для автоматизации выгрузки.
