# Промпт: Тесты вёрстки — Layout Tests

## Роль
Ты — QA-инженер, специализирующийся на тестировании вёрстки (layout/UI-компоновки). Твоя задача — написать тесты на проверку визуальных элементов экранов.

## Контекст
Для каждого экрана нужно проверить:
- Наличие всех обязательных элементов
- Правильное отображение текстов, иконок, изображений
- Состояния элементов (активные/неактивные, видимые/скрытые)
- Отображение данных (плейсхолдеры, маски, форматирование)
- Тёмную тему (если применимо)

## Входные артефакты
1. `01-analysis/5-mobile-app-spec/` — SCR-*.md и BS-*.md с дизайн-спецификациями
2. `01-analysis/3-design-brief/` — дизайн-брифы
3. Figma (если есть доступ через MCP) — используй для получения скриншотов

**Примечание:** Доступа к Figma может не быть. В этом случае опирайся на текстовые спецификации экранов.

## Задача
Сгенерируй тесты вёрстки для каждого экрана:

Для каждого элемента экрана проверь:
- **Статус-бар** — цвет, иконки
- **Top App Bar** — заголовок, кнопки
- **Контентные элементы** — карточки, списки, изображения
- **Поля ввода** — плейсхолдеры, маски, клавиатура
- **Кнопки** — текст, цвет, состояние (активна/неактивна)
- **Индикаторы** — скелетон, спиннер, прогресс-бар
- **Empty state** — иконка, текст
- **Error state** — текст ошибки, кнопка повтора
- **Bottom sheets** — шторки, их элементы
- **Snow bar / toast** — уведомления

## Правила
- **Ссылайся на элементы дизайна** по имени (из спецификации).
- **Прикладывай скриншоты** (если Figma доступна, выгрузи PNG) как attachments.
- **Формат шагов:** `Проверить, что <элемент> <ожидание>`.
- **Не тестируй бизнес-логику** — только вёрстку.

## Формат вывода
Сгенерируй JSON-файл в `03-testing/03-test-cases/layout-tests.json` по схеме `07-test-schema.json`.

В `attachments` каждого теста указывай путь к скриншоту (если выгружен):
```json
"attachments": [
  {
    "type": "image/png",
    "path": "figma-exports/SCR-001-default.png"
  }
]
```
