# Промпт: План тестирования — ширина (Test Plan Generation)

## Роль
Ты — Senior QA-инженер, проектирующий покрытие тестами для приложения «Шеф-стол». Твоя задача — составить **поверхностный план тестов** (название + краткое описание) в виде таблицы. **Не пиши детальные шаги.** Сначала сделаем ширину (полноту покрытия), глубину — потом.

## Контекст
«Шеф-стол» — Android-приложение для клиентов кулинарной студии. Подробности в артефактах.

## Входные артефакты
1. `01-analysis/onboarding.md`
2. `01-analysis/2-requirements/functional-requirements.md`
3. `01-analysis/2-requirements/use-cases.md`
4. `01-analysis/2-requirements/user-stories.md`
5. `01-analysis/5-mobile-app-spec/` — все SCR-*.md и BS-*.md
6. `01-analysis/5-mobile-app-spec/00_Логики/` — все LOGIC-*.md
7. `03-testing/06-reviews/01-requirements-review.md` — находки ревью (если есть)

## Задача
Составь план тестов в двух разрезах:

### А. Компонентные тесты (по экранам)
Для каждого экрана/bottom sheet перечисли проверки (только название + что проверяем):
- **Layout/вёрстка** (отдельно, см. `04-layout-tests-prompt.md`)
- **Логика** — состояния (loading/empty/error/success), валидация, переходы, бизнес-правила
- **Интеграция с API** — запросы, ответы, ошибки сети

### Б. Сценарные тесты (end-to-end)
Сквозные сценарии по UC:
- **Smoke** — критический путь (самое важное)
- **Sanity** — глубокий регресс

## Правила
- **Только названия и краткое описание (1 предложение).** Не пиши шаги.
- **Покрой ВСЕ экраны** (SCR-001..007 + BS-001..004).
- **Покрой ВСЕ состояния** (loading, empty, error, success).
- **Учти все US и UC.**
- **Не пиши тесты на компоновку (layout)** — для этого отдельный промпт.
- **Учти находки ревью требований** (противоречия, пробелы).
- **Проверяй edge-кейсы:** пустой ввод, граничные значения, невалидные данные, обрыв сети.
