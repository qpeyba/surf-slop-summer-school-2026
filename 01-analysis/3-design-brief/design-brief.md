# Постановка на дизайн — «Шеф-стол»

> Документ для дизайнера: описывает продукт, аудиторию, структуру экранов и требования к макетам.
> Основан на слое требований: `../2-requirements/`.

---

## 1. О продукте

**«Шеф-стол»** — мобильное приложение (Android, нативное) для самостоятельной записи клиентов на кулинарные мастер-классы.

**Проблема:** сейчас запись идёт через WhatsApp и Google-таблицы — двойные бронирования, потеря продуктов, хаос у владельца.

**Решение:** клиент сам выбирает класс в расписании, бронирует место, отменяет или переносит запись. Владелец только мониторит.

**Бизнес-требования:** [BR-1–BR-4](../2-requirements/business-requirements.md)

---

## 2. Целевая аудитория

- **Клиенты студии** — мужчины и женщины 25–45 лет, интересуются кулинарией, хотят удобно записаться с телефона
- Не требуют обучения — интерфейс должен быть интуитивным
- Основной сценарий: открыл → выбрал класс → записался за < 1 минуты

## 3. Объём работ

### 3.1. Основные экраны (7 шт.)

| ID | Экран | Ключевая цель |
|----|-------|---------------|
| SCR-001 | Расписание | Найти класс и перейти к записи |
| SCR-002 | Карточка класса | Изучить детали и принять решение |
| SCR-003 | Бронирование | Выбрать опции и подтвердить запись |
| SCR-004 | Успешная запись | Увидеть подтверждение и детали брони |
| SCR-005 | Мои брони | Управлять активными и просматривать историю |
| SCR-006 | Авторизация | Войти по номеру телефона (OTP) |
| SCR-007 | Профиль | Управлять аллергиями, видеть баланс и статус |

### 3.2. Шторки / модальные окна (4 шт.)

| ID | Шторка | Ключевая цель |
|----|--------|---------------|
| BS-001 | Отмена брони | Подтвердить отмену, увидеть сумму возврата |
| BS-002 | Перенос брони | Выбрать новый слот для переноса |
| BS-003 | Оценка шефа | Поставить звёзды и написать отзыв |
| BS-004 | Фильтр дат | Выбрать дату для фильтрации расписания |

## 4. Технические требования к макетам

### 4.1. Платформа

- **Android, нативное приложение** — дизайн в Material Design 3 (Material You)
- **Формат:** Figma, Auto Layout, Components + Variants
- **Разрешение:** базовый фрейм 412×915 dp (Pixel 7 / современный Android)

### 4.2. Обязательные состояния каждого экрана

См. [00-foundations.md](00-foundations.md), раздел 3. Каждый экран с данными от API должен быть показан в состояниях:
- Loading (скелетон)
- Empty (заглушка)
- Error (ошибка + retry)
- Success (контент)

### 4.3. Трассировка

Каждый экран спецификации (SCR-*) содержит ссылки на конкретные FR, UC, US. Дизайнер должен убедиться, что:
- Каждый функциональный элемент экрана покрывает требование
- Нет элементов, не выведенных из требований
- Нет функций Phase 2 в MVP-экранах

### 4.4. Что НЕ входит в MVP (Phase 2)

- Онлайн-оплата — вместо неё кнопка-заглушка «Скоро: онлайн-оплата» (FR-3.3)
- Рейтинги шефов до записи (рейтинг виден — FR-1.4, но система рейтингов бэкенда уже существует)
- Интерфейсы шефа и администратора (NFR-4)
- Пакетная запись на несколько классов (Q1)
- Лист ожидания (Q4)

## 5. Фирменный стиль

Студия «Шеф-стол» — лофт на бывшем заводе:
- **Атмосфера:** индустриальная + уютная кухня (кирпич, дерево, металл, тёплый свет)
- **Настроение:** гастрономическое приключение, а не формальный урок
- **Рекомендации дизайнеру:**
  - Цветовая палитра: тёплые природные тона (терракота, оливковый, сливочный) + контрастный тёмный текст
  - Типографика: чистый гротеск без засечек
  - Фотографии: крупные, аппетитные фото блюд — ключевой визуальный элемент карточек классов
  - Иконография: минималистичная, линейная (Material Symbols)

## 6. Критерии приёмки дизайна

- [ ] Все 7 экранов и 4 шторки замакетированы
- [ ] Каждый экран имеет состояния: loading, empty, error, success (где применимо)
- [ ] Каждый экран связан с требованиями (трассировка по ID)
- [ ] Нет «осиротевших» UI-элементов (не выведенных из FR/UC/US)
- [ ] Нет функций Phase 2 в макетах
- [ ] Доступность: контраст, touch targets, content descriptions
- [ ] Все экраны используют единые токены из 00-foundations.md
- [ ] Навигационные переходы соответствуют карте из реестра экранов
- [ ] Тап по push-уведомлению ведёт на правильный экран
