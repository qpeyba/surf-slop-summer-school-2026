# 00-foundations — Общие правила дизайна

> Фиксирует сквозные правила UI: токены, навигацию, состояния, доступность.
> Все поэкранные спецификации (SCR-*) и шторки (BS-*) ссылаются на этот документ — не дублируют описанные здесь паттерны.

---

## 1. Дизайн-токены

| Токен | Значение | Применение |
|-------|----------|------------|
| Основной цвет (Primary) | Уточняется дизайнером (рекомендация: тёплый терракотовый / оранжевый — ассоциация с кулинарией) | Кнопки CTA, активные элементы, акценты |
| Акцентный цвет (Accent) | Уточняется дизайнером (зелёный — успех, красный — ошибка/отмена) | Статусы, бейджи, индикаторы |
| Фон экрана | `#FAFAFA` (светло-серый) | Все экраны |
| Фон карточки | `#FFFFFF` (белый) | Карточки классов, броней |
| Текст основной | `#1A1A1A` | Заголовки, основной текст |
| Текст второстепенный | `#757575` | Подписи, мета-информация |
| Радиус скругления карточек | 12 dp | Карточки классов, броней, профиля |
| Отступы экрана | 16 dp по горизонтали | Все экраны |
| Высота кнопок | 48 dp | Все CTA |
| Шрифт | System default (Roboto для Android) | Всё приложение |
| Длительность класса (`CLASS_DURATION`) | ~3 часа | Отображается в карточке класса (SCR-001) и деталях (SCR-002). Источник: `4-design/data-model.md` §2.3 — `Slot.dateTime` «длительность ~3 часа подразумевается». Не хардкодить в каждом экране — ссылаться на эту константу |

## 2. Навигация — Tab Bar

Приложение имеет три вкладки в нижнем Tab Bar (видны только авторизованному пользователю):

| Позиция | Иконка | Название | Экран |
|---------|--------|----------|-------|
| 1 | Иконка календаря | Расписание | SCR-001_Schedule |
| 2 | Иконка списка/билета | Мои брони | SCR-005_MyBookings |
| 3 | Иконка профиля | Профиль | SCR-007_Profile |

**Правила:**
- Tab Bar закреплён снизу, виден на всех трёх экранах
- При push-переходах (SCR-001 → SCR-002 → SCR-003 → SCR-004) Tab Bar скрывается
- Возврат назад — системная кнопка Back (стрелка) в Top App Bar
- При переходе на экран в другой вкладке — состояние текущей вкладки сохраняется

## 3. Паттерны состояний экрана

Каждый экран, загружающий данные асинхронно, должен поддерживать четыре состояния:

### 3.1. Loading (Загрузка)

- **Что видит пользователь:** скелетон (shimmer-эффект) повторяющий форму будущего контента: прямоугольники на месте карточек, круг на месте аватара
- **Когда:** первая загрузка данных с API (расписание, история броней, профиль)
- **Таймаут:** если загрузка > 10 секунд — показывать сообщение «Загрузка занимает больше времени, чем обычно»

### 3.2. Empty (Пусто)

- **Что видит пользователь:** иконка-иллюстрация по центру экрана + пояснительный текст
- **Тексты:**
  - Расписание пусто: «Пока нет доступных классов» (FR-1.3)
  - Брони пусты: «У вас пока нет броней» (UC-5, альт. 1а)
  - История пуста: «За последние 3 месяца броней не найдено»
- **Действие:** на расписании — кнопка «Обновить»; на бронях — кнопка «Перейти к расписанию»

### 3.3. Error (Ошибка)

- **Что видит пользователь:** иконка ошибки + текст «Что-то пошло не так» + описание ошибки (если есть от API) + кнопка «Повторить»
- **Когда:** ошибка сети, таймаут, 5xx от API, 401 (требуется повторная авторизация)
- **401 отдельно:** редирект на SCR-006_Auth с сообщением «Сессия истекла, войдите снова»

### 3.4. Success (Успех / Данные)

- **Что видит пользователь:** контент экрана согласно спецификации SCR-*
- **Pull-to-refresh:** поддерживается на SCR-001 (расписание) и SCR-005 (брони)

## 4. Push-уведомления (системные)

Приложение принимает push-уведомления от бэкенда (FR-5.1, FR-5.2):

| Событие | Заголовок | Тело | Действие по тапу |
|---------|-----------|------|------------------|
| Напоминание (24ч/3ч/30мин) | «Скоро начнётся: [программа]» | «[Дата], [Время] • [Адрес студии]» | Открыть SCR-005_MyBookings, активная бронь |
| Отмена студией | «Класс отменён» | «[Программа] [Дата] отменён. [Причина]» | Открыть SCR-005_MyBookings, отменённая бронь |

Canvas-уведомления внутри приложения: snackbar/toast для кратких подтверждений («Бронь создана», «Бронь отменена», «Оценка сохранена»).

## 5. Доступность (Accessibility)

Минимальный уровень: **WCAG 2.1 AA**.

| Требование | Реализация |
|------------|------------|
| Контраст текста | Минимум 4.5:1 для обычного текста, 3:1 для крупного |
| Touch targets | Минимум 48×48 dp для всех интерактивных элементов |
| Content descriptions | Все иконки и изображения имеют `contentDescription` |
| Звёзды рейтинга | Дублируются текстовой меткой («4 из 5») |
| Кнопки-заглушки | Имеют состояние `disabled` + `contentDescription` «Недоступно» |
| Ошибки валидации | Отображаются текстом под полем, а не только цветом |

---

## 6. Кнопка «Назад» и жесты

- **Аппаратная кнопка «Назад» (Android):** навигация назад по стеку экранов
- **Свайп назад (жест Android):** поддерживается системно
- **Стрелка «Назад» в Top App Bar:** на всех экранах глубже Tab Bar
- **Закрытие шторки (BS-*):** свайп вниз или тап по затемнённой области

---

## 7. Стиль иллюстраций

- **Empty state:** линейные иллюстрации в стиле приложения, нейтральные, не перегруженные
- **Иконки:** Material Design Icons (стандартный набор)
- **Фото блюд:** загружаются из API, placeholder на время загрузки — серый прямоугольник со скруглением 8 dp и иконкой фотоаппарата по центру
