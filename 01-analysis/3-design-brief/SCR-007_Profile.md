# SCR-007 — Профиль

| Поле | Значение |
|------|----------|
| **Название** | Профиль |
| **Тип** | Основной экран, Tab 3 |
| **FR** | FR-2.2, FR-2.3, FR-2.4 |
| **UC** | — (UI-экран) |
| **US** | US-2, US-7, US-11 |

---

## Цель экрана

Показать клиенту его профиль: телефон, аллергии, балльный баланс, статус лояльности. Дать возможность редактировать аллергии и перейти к истории броней.

---

**Что видит пользователь:**

1. **Top App Bar** — заголовок «Профиль»
2. **Блок профиля** — аватар (placeholder — иконка шеф-колпака или инициалы), номер телефона, статус лояльности (если «Постоянный» — бейдж со звездой)
3. **Блок «Лояльность»** (FR-2.3):
   - Баланс баллов: «1 250 баллов» с иконкой
   - Прогресс-бар до следующего уровня (если не постоянный) или подпись «Постоянный клиент»
   - Статус: «Постоянный клиент» с бейджем и пояснением «5 классов за 2 месяца» (domain 4.8, строка 166)
   - Если статус не достигнут: «N из 3 классов до статуса «Постоянный»»
4. **Блок «Пищевые аллергии»** (FR-2.2):
   - Список аллергий — чипы с кнопкой удаления «✕»
   - Кнопка «+ Добавить аллергию» — при тапе поле ввода (свободный текст или выбор из списка)
   - Если список пуст — текст «Не указаны» + кнопка «Добавить»
5. **Пункт «Мои брони»** — строка-ссылка со стрелкой «→», переход на SCR-005
6. **Кнопка «Выйти»** — текстовая кнопка (outline или текст), завершение сессии, возврат на SCR-006

---

## Редактирование аллергий (inline)

При тапе «+ Добавить аллергию»:
- Появляется текстовое поле ввода с кнопкой «Добавить»
- После ввода — новый чип в списке
- Удаление — тап по «✕» на чипе, без подтверждения
- Изменения сохраняются сразу (автосохранение на бэкенд)

---

## Состояния экрана

### Loading
Скелетон: круг для аватара, строка для телефона, блок с 3 строками для лояльности, 2 строки для аллергий.

### Error
«Не удалось загрузить профиль» + кнопка «Повторить».

### Success
Как в раскадровке.
