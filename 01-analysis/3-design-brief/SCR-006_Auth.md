# SCR-006 — Авторизация

| Поле | Значение |
|------|----------|
| **Название** | Авторизация |
| **Тип** | Стартовый экран (показывается, если нет сессии) |
| **FR** | FR-2.1 |
| **UC** | — |
| **US** | US-1 |

---

## Цель экрана

Авторизовать клиента по номеру телефона через SMS-код. Экран показывается при первом запуске и при истечении сессии.

---

## Раскадровка

### Шаг 1: Ввод номера телефона

```
┌─────────────────────────────────┐
│                                 │
│                                 │
│        [Логотип «Шеф-стол»]     │
│                                 │
│     Добро пожаловать!           │
│     Войдите, чтобы записаться   │
│     на кулинарные классы        │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 🇷🇺 +7 │ 999 123 45 67  │  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │     Получить код          │  │
│  └───────────────────────────┘  │
│                                 │
│  ⓘ Нажимая «Получить код»,     │
│  вы соглашаетесь с условиями    │
│                                 │
└─────────────────────────────────┘
```

**Что видит пользователь (шаг 1):**

1. **Логотип** — название студии, возможно иконка/шеф-колпак (по центру экрана сверху)
2. **Приветственный текст** — «Добро пожаловать! Войдите, чтобы записаться на кулинарные классы»
3. **Поле ввода телефона** — предзаполненный код страны «+7», поле для 10 цифр номера, маска ввода: `+7 (XXX) XXX-XX-XX`
4. **Кнопка «Получить код»** — активна только когда введено 10 цифр номера. На время отправки — лоадер внутри кнопки
5. **Текст согласия** — мелким шрифтом, «Нажимая «Получить код», вы соглашаетесь с условиями»

### Шаг 2: Ввод SMS-кода (OTP)

```
┌─────────────────────────────────┐
│  ← Назад                        │
├─────────────────────────────────┤
│                                 │
│     Код отправлен               │
│     на +7 (999) 123-45-67       │
│                                 │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐      │
│  │ 4 │ │ 8 │ │   │ │   │      │
│  └───┘ └───┘ └───┘ └───┘      │
│         (курсор на 3-й цифре)   │
│                                 │
│  Отправить код повторно         │
│  через 0:42                     │
│                                 │
│  (при ошибке)                   │
│  ⚠ Неверный код, попробуйте     │
│    ещё раз                      │
│                                 │
└─────────────────────────────────┘
```

**Что видит пользователь (шаг 2):**

1. **Top App Bar** — «← Назад» (вернуться к вводу номера)
2. **Текст** — «Код отправлен на +7 (XXX) XXX-XX-XX»
3. **4 поля для цифр** — отдельные квадраты, ввод по одной цифре, автофокус на первом поле, автоматический переход к следующему. При вводе 4-й цифры — автоматическая отправка
4. **Таймер повторной отправки** — «Отправить код повторно через N:NN». Когда таймер истекает — текст становится ссылкой «Отправить код повторно»
5. **Ошибка** (при неверном коде) — текст ошибки под полями: «Неверный код, попробуйте ещё раз», поля очищаются, фокус на первое

---

## Валидация и ошибки

| Ситуация | Поведение |
|----------|-----------|
| Неверный формат телефона (< 10 цифр) | Кнопка «Получить код» неактивна |
| Ошибка сети при отправке SMS | Текст под полем: «Не удалось отправить код. Проверьте подключение к интернету» |
| Неверный код | Ошибка под полями, поля очищаются, счётчик попыток (3 попытки, после — блокировка на 60 сек) |
| Таймаут кода (истёк срок действия) | «Срок действия кода истёк. Запросите новый» |
| 401 — сессия истекла (в процессе использования) | Экран авторизации с сообщением «Сессия истекла, войдите снова» |

---

## Состояния экрана

### Шаг 1 — Отправка кода (Loading)
Кнопка меняется на лоадер (spinner), поле телефона блокируется.

### Шаг 2 — Проверка кода (Loading)
Рамки полей меняют цвет на нейтральный, показывается индикатор загрузки. При успехе — плавный переход на Tab Bar (SCR-001).

### Error
Встроенная ошибка под полями, не отдельный экран.
