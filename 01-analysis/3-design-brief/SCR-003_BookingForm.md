# SCR-003 — Бронирование

| Поле | Значение |
|------|----------|
| **Название** | Форма бронирования |
| **Тип** | Экран (push из SCR-002) |
| **FR** | FR-3.1, FR-3.2, FR-3.3 |
| **UC** | UC-1 (шаги 4–5) |
| **US** | US-5, US-6 |

---

## Цель экрана

Собрать подтверждение опций бронирования: экипировка (своя / прокат), аллергии (из профиля), отобразить итоговую стоимость и кнопку подтверждения с заглушкой онлайн-оплаты.

---

**Что видит пользователь:**

1. **Top App Bar** — «← Назад», заголовок «Бронирование»
2. **Блок «Краткая информация»** — компактная карточка класса: программа, дата/время, шеф с рейтингом. Напоминание, на что именно записывается клиент.
3. **Блок «Экипировка»** — radio buttons (FR-3.2):
   - «Своя экипировка» — клиент приходит со своим фартуком и ножами
   - «Прокат экипировки» — единый набор (фартук + ножи), бесплатно. По умолчанию выбрано значение из профиля (domain 2.3, строка 64)
4. **Блок «Пищевые аллергии»** (FR-2.2, FR-3.2) — список аллергий, подтянутых из профиля:
   - Каждая аллергия — чип с кнопкой удаления «✕»
   - Кнопка «+ Добавить аллергию» — при тапе поле ввода с автодополнением (или свободный ввод)
   - Если аллергий нет — текст «Не указаны» + кнопка «Добавить»
   - Пояснение мелким шрифтом: «Аллергии будут переданы студии для учёта при подготовке»
5. **Блок «Итого»** — расчёт стоимости (FR-3.3):
   - Стоимость класса
   - Стоимость проката (0 ₽, бесплатно)
   - Разделительная линия
   - Итоговая сумма (жирным)
6. **Кнопка «Подтвердить запись»** — основная CTA, закреплена снизу
7. **Кнопка-заглушка «Скоро: онлайн-оплата»** (FR-3.3) — под основной кнопкой, стиль outline, состояние disabled, не нажимается. Рядом иконка 🔒 или часов.
8. **Информационная строка** — «Оплата при личном визите» с иконкой info

## Валидация перед отправкой

| Проверка | Действие при ошибке |
|----------|---------------------|
| Нет интернета | Сообщение «Нет подключения к интернету», кнопка не блокируется, но при тапе — ошибка сети |
| Класс уже заполнен (409 от API) | Сообщение «Мест нет, запись невозможна», обновить счётчик мест |
| Класс уже начался (410 от API) | Сообщение «Запись на класс закрыта», кнопка блокируется |
| Уже есть бронь на это время (409) | «У вас уже есть бронь на это время» |

---

## Состояния экрана

### Loading (при загрузке данных профиля)
Скелетон: компактная карточка, placeholder для radio buttons, 2 строки для аллергий.

### Error
«Не удалось загрузить данные профиля» + «Повторить». Кнопка «Подтвердить запись» остаётся активной (аллергии и экипировка не критичны для записи).

### Success
Как в раскадровке.
