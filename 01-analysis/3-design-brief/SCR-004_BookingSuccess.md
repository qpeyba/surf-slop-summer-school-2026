# SCR-004 — Успешная запись

| Поле | Значение |
|------|----------|
| **Название** | Успешная запись |
| **Тип** | Экран подтверждения (push из SCR-003) |
| **FR** | FR-3.3, FR-5.1 |
| **UC** | UC-1 (шаг 8) |
| **US** | US-5 |

---

## Цель экрана

Подтвердить клиенту, что бронь создана успешно. Показать ключевые детали брони и дать дальнейшие действия: перейти к броням или вернуться в расписание.

---

**Что видит пользователь:**

1. **Иконка успеха** — крупная галочка в круге, зелёный цвет, анимация появления (scale + fade)
2. **Заголовок** — «Вы записаны!» крупным шрифтом
3. **Карточка брони** — повторяет структуру карточки из SCR-005, содержит:
   - Название программы
   - Дата и время (полная дата + время)
   - Имя шефа
   - Адрес студии
   - Тип экипировки (Своя / Прокат)
   - Список аллергий (если указаны)
   - Итоговая стоимость + пометка «оплата при визите»
4. **Информационный блок о напоминаниях** (FR-5.1) — текст «Напоминания придут: за 24 ч, 3 ч и 30 мин до начала» с иконкой часов
5. **Кнопка «Мои брони»** — transition to SCR-005_MyBookings (Tab 2)
6. **Текстовая ссылка «Вернуться к расписанию»** — transition to SCR-001 (Tab 1)

---

## Состояния экрана

Экран статический — загружается после успешного ответа API. Дополнительных состояний (loading/empty/error) не требует. Если экран открыт, а бронь была отменена студией в этот момент — push-уведомление, экран остаётся (бронь уже в статусе «Активна» на момент показа).

## Навигация

| Откуда | Событие | Куда |
|--------|---------|------|
| SCR-003 | Успешный ответ API (201) | SCR-004 |
| SCR-004 | Тап «Мои брони» | SCR-005_MyBookings (Tab 2) |
| SCR-004 | Тап «Вернуться к расписанию» | SCR-001_Schedule (Tab 1) |
| SCR-004 | Аппаратный «Назад» | SCR-001 (не возвращаться на SCR-003 — форма уже отправлена) |

**Важно:** SCR-004 заменяет SCR-003 в стеке навигации (нельзя вернуться назад и повторно отправить форму).
