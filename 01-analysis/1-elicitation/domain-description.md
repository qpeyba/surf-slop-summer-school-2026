# Описание предметной области — «Кулинарная студия»

> Составлено на основе брифа заказчика и уточнений по скоупу.  
> Описывает домен «как есть» (AS-IS) — без привязки к конкретному техническому решению.

---

## 1. Общая картина

**Кулинарная студия «Шеф-стол»** — офлайн-бизнес, проводящий групповые кулинарные мастер-классы в помещении лофта на территории бывшего завода. Студия работает третий год. Основная проблема — ручное управление записью клиентов через WhatsApp и Google-таблицы, приводящее к ошибкам двойного бронирования, неучтённым отменам и потере продуктов.

**Ключевая цель автоматизации:** перевести запись клиентов в самообслуживание через мобильное приложение, оставив владельцу только мониторинг и ручные корректировки в исключительных ситуациях.

## 2. Ключевые сущности предметной области

### 2.1. Кулинарный класс (Class / Slot)

Одно событие в расписании студии. Характеризуется:

| Атрибут | Описание |
|---------|----------|
| Дата и время начала | Фиксированное; класс длится ~3 часа |
| Программа / меню | Что именно готовят (напр. «Итальянская паста», «Сложная выпечка»), фото блюд |
| Уровень сложности | Для новичков / для опытных |
| Шеф | Кто ведёт класс; отображается рейтинг шефа |
| Вместимость | До 12 рабочих мест; до 8 — для классов с духовками/сложной техникой |
| Стоимость | Цена участия, показывается клиенту |
| Адрес студии | Отображается в карточке класса |
| Статус | Активен / Отменён студией (форс-мажор) |

**Правила:**
- Расписание формируется на неделю вперёд (7 дней), отображается по умолчанию.
- Клиент видит точное число оставшихся мест (напр. «Осталось 3 из 12»).
- Запись возможна не позднее чем за 10 минут до начала.
- Фильтрация расписания: **только по дате**.
- Класс может быть отменён студией целиком при срыве поставки продуктов или поломке оборудования → все брони получают статус «Отменён студией», клиентам уходит push-уведомление.
- Повторная запись на отменённый студией слот запрещена.
- Лист ожидания отсутствует. Если мест нет — показывается «Мест нет».

### 2.2. Шеф (Chef)

Сотрудник, ведущий класс.

| Атрибут | Описание |
|---------|----------|
| Имя | Идентификация для клиентов |
| Статус | Постоянный (5 чел.) / Приглашённый (сезонно) |
| Рейтинг | Агрегированная оценка от клиентов (5 звёзд), видна в карточке класса до записи |
| Специализация | Подразумевается (ведёт определённые программы) |

### 2.3. Клиент (Client)

Конечный пользователь мобильного приложения — тот, кто записывается на классы.

| Атрибут | Описание |
|---------|----------|
| Телефон | Идентификатор; регистрация только по SMS |
| Аллергии | Список пищевых ограничений; заполняется один раз в профиле, редактируемо |
| Балльный баланс | Начисляется при no-show (денежный эквивалент), пополняется по программе лояльности |
| Статус лояльности | «Постоянный» при 3+ посещённых классах за последние 2 месяца |
| История броней | Прошлые и текущие записи за последние 3 месяца |
| Своя экипировка | Флаг: есть ли свой фартук и набор ножей |

### 2.4. Бронь (Booking)

Связь клиента с конкретным классом.

| Атрибут | Описание |
|---------|----------|
| Клиент | Кто записался |
| Класс | На какое событие |
| Дата и время создания | Когда записался |
| Статус | Активна / Завершена / Отменена клиентом / Отменена студией / Клиент не пришёл |
| Экипировка | Со своим набором или прокат (единый набор: фартук + ножи) |
| Оценка шефу | 5 звёзд + опциональный текст; можно редактировать в любой момент |

**Правила:**
- Гарантия «0 двойных броней» — на стороне бэкенда (атомарная проверка свободных мест при бронировании). Клиентское приложение полагается на ответ API.
- Отмена клиентом: **бесплатно за 12+ часов до начала; за 12-0 часов — возврат 50%.** Возврат — ручной процесс владельца (наличные/перевод), приложение только фиксирует сумму к возврату.
- Перенос брони на другой слот: поддерживается одним действием (кнопка «Перенести»).
- No-show (клиент не пришёл): **начисление баллов на баланс** в денежном эквиваленте.
- При отмене студией бронь не удаляется, а меняет статус на «Отменена студией».

### 2.5. Прокатная экипировка (Equipment Rental)

Фартук и набор ножей, предоставляемые студией напрокат.

| Атрибут | Описание |
|---------|----------|
| Тип | Единый набор: фартук + ножи (не дробится) |
| Стоимость проката | **Бесплатно** (включено в стоимость класса) |
| Доступный фонд | **Не ограничен** — экипировки всегда хватает на всех |

## 3. Роли и зоны ответственности

| Роль | Ответственность | Интерфейс |
|------|----------------|-----------|
| **Клиент** | Просмотр расписания, запись, отмена, оценка шефа, управление профилем | **Мобильное приложение** (текущая поставка) |
| **Шеф** | Просмотр своего расписания, списка участников | Существующая инфраструктура (вне скоупа) |
| **Владелец / Администратор** | Управление расписанием, ручные корректировки, мониторинг | Существующая админка (вне скоупа) |

## 4. Ключевые бизнес-процессы (целевые)

### 4.1. Запись на класс

1. Клиент открывает приложение → видит слоты на ближайшие 7 дней (фильтр только по дате).
2. Выбирает класс → видит полную карточку: фото блюд, меню, адрес, рейтинг шефа, уровень сложности, стоимость, точное число свободных мест.
3. Указывает: своя экипировка или прокат (единый набор, бесплатно). Аллергии подтягиваются из профиля.
4. Подтверждает запись → приложение отправляет запрос к API бэкенда.
5. Бэкенд атомарно проверяет и бронирует место → возвращает успех или отказ (мест нет).
6. Клиент видит подтверждение, бронь появляется в его списке.
7. За 24ч, 3ч и 30 мин до начала — push-напоминания.

**Ограничение:** запись закрывается за 10 минут до начала класса.

### 4.2. Отмена клиентом

1. Клиент находит активную бронь → нажимает «Отменить».
2. Приложение проверяет время до начала:
   - **>12 часов:** бесплатная отмена.
   - **≤12 часов:** возврат 50%, информирует клиента.
3. Бронь переходит в статус «Отменена клиентом», место освобождается.
4. Информация о сумме к возврату фиксируется в приложении; **сам возврат — ручной процесс владельца** (наличные/перевод).

### 4.3. Перенос брони

1. Клиент находит активную бронь → нажимает «Перенести».
2. Выбирает новый слот из доступных.
3. Старая бронь отменяется, создаётся новая — одним действием.

### 4.4. No-show (клиент не пришёл)

1. После фактического начала класса брони, по которым клиент не явился, получают статус «Клиент не пришёл».
2. Клиенту начисляются баллы на баланс в денежном эквиваленте.

### 4.5. Отмена студией (форс-мажор)

1. Администратор в админке (вне скоупа) отменяет класс с указанием причины.
2. Все брони этого класса получают статус «Отменён студией».
3. Клиенты получают push-уведомление.
4. Повторная запись на этот слот заблокирована.

### 4.6. Оценка шефа

1. После завершения класса клиент получает предложение оценить шефа.
2. Выставляет оценку: 5 звёзд + опциональный текстовый отзыв.
3. Оценку можно редактировать в любой момент.
4. Оценка агрегируется в рейтинг шефа, видимый в карточке класса до записи.

### 4.7. Уведомления (только push)

| Событие | Когда |
|---------|-------|
| Напоминание о классе | За 24 часа, за 3 часа, за 30 минут до начала |
| Отмена класса студией | Немедленно при отмене |

### 4.8. Программа лояльности

- **Балльная система:** 1 класс = N баллов, баллы можно тратить на оплату.
- **Статус «постоянный клиент»:** 3+ посещённых класса за последние 2 месяца.
- При no-show также начисляются баллы на баланс (денежный эквивалент).

## 5. Ограничения и допущения

| # | Ограничение |
|---|-------------|
| R1 | Бэкенд — black-box источник истины. Все критические проверки (свободные места, двойное бронирование) — на его стороне. |
| R2 | Клиентское приложение не создаёт и не редактирует слоты, программы, шефов — только потребляет через API. |
| R3 | Текущая поставка — только роль Клиент. Интерфейсы шефа и админа уже существуют. |
| R4 | Горизонт отображения расписания по умолчанию — 7 дней от текущего момента. |
| R5 | Если расписания нет — клиенту показывается empty state: «Пока нет доступных классов». |
| R6 | Контракт API — каноническая схема данных (исторических данных / легаси нет). |
| R7 | Проект учебный/тестовый. |
| R8 | Бюджет ограничен, приоритет — MVP к старту сезона (~2 месяца). |
| R9 | Платформа: **только Android**, нативное приложение из Google Play. |
| R10 | Онлайн-оплата отсутствует. Показывается итоговая цена, кнопка-заглушка «Скоро: онлайн-оплата». |
| R11 | Возвраты при отмене — ручной процесс владельца; приложение только фиксирует сумму к возврату. |
| R12 | Прокатная экипировка: фонд не ограничен, бесплатно, единый набор (фартук + ножи). |
