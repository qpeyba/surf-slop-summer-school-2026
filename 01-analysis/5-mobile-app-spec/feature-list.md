# Feature List — «Шеф-стол»

> Скоуп поставки MVP: слой функциональности клиентского мобильного приложения.
> Бэкенд — black-box источник истины (NFR-3). Интерфейсы шефа и администратора — вне скоупа (NFR-4).

---

## MoSCoW-категоризация

### Must Have (MVP Phase 1)

| # | Фича | FR | Описание |
|---|------|----|----------|
| 1 | Просмотр расписания на 7 дней | FR-1.1, FR-1.3 | Расписание с фильтрацией по умолчанию на текущую дату. Empty state: «Пока нет доступных классов» |
| 2 | Фильтрация расписания по дате | FR-1.2 | Выбор одной даты в календаре, диапазон 7 дней от текущей |
| 3 | Карточка класса | FR-1.4, FR-4.3 | Дата/время, меню + фото, сложность, шеф + рейтинг, свободные места (`capacity − bookedCount`), цена, адрес, статус. Заблокирована запись на отменённые слоты |
| 4 | Бронирование места на класс | FR-3.1, FR-3.2 | Не позднее 10 мин до начала. Выбор экипировки (своя/прокат), аллергии из профиля. Pre-check доступности (клиент) + атомарная проверка (бэкенд). Заглушка онлайн-оплаты |
| 5 | Отображение итоговой стоимости + заглушка оплаты | FR-3.3 | Кнопка «Записаться» с ценой, после подтверждения — экран успеха с заглушкой «Скоро: онлайн-оплата» |
| 6 | Экран успешного бронирования | FR-3.3 | Подтверждение записи, переход в «Мои брони» или на Tab Bar |
| 7 | OTP-авторизация по телефону | FR-2.1 | Номер телефона → SMS-код → Bearer-токен. 401 → редирект на авторизацию |
| 8 | Отмена активной брони | FR-4.1 | Правило 12ч: >12ч бесплатно, ≤12ч — возврат 50% (фиксирует бэкенд). Предупреждение с суммой возврата |
| 9 | Перенос брони на другой слот | FR-4.2 | Выбор слота из расписания, атомарная транзакция (отмена+создание). Наследование equipmentType |
| 10 | Push-напоминания о классе | FR-5.1 | За 24ч, 3ч, 30мин. Тап → SCR-005, активная бронь |
| 11 | Push-уведомление об отмене студией | FR-5.2 | Мгновенно. Тап → SCR-005, отменённая бронь |

## Нефункциональный бэклог

| ID | Категория | Реализация в MVP |
|----|-----------|------------------|
| NFR-1 | Атомарность бронирования | Доверие бэкенду. Pre-check на клиенте — UX-помощник, не гарантия |
| NFR-2 | Только Android | Нативное приложение, Google Play, minSdk уточняется |
| NFR-3 | Бэкенд — black-box | Вся бизнес-логика на бэкенде. Приложение — потребитель API |
| NFR-4 | Только роль «Клиент» | Шеф и админ — вне скоупа |
| NFR-5 | Каноническая схема API | Без легаси-контрактов |
