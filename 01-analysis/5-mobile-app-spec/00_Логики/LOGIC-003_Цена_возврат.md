## Обзор

Определяет отображение стоимости участия и сумму возврата при отмене. Цена слота (`slot.price`) приходит из API и не хардкодится. Сумма возврата (`refundAmount`) рассчитывается бэкендом как `price × 0.5` при отмене ≤12ч до начала и возвращается в ответе `cancelBooking` / `listBookings`. Приложение только отображает оба значения — не вычисляет.

### Бизнес-ценность

- Прозрачность для клиента: цена и сумма возврата видны до подтверждения действия
- Цена — из ответа API, не зашита в клиенте
- refundAmount фиксируется бэкендом, приложение доверяет этому значению

## Описание логики

### Цена участия

```
цена = slot.price  (из ответа API: GET /slots, GET /slots/{slotId})
```

Отображается в рублях, формат: `{price} ₽`. Источник: поле `price` в объекте `Slot`.

### Сумма возврата

```
refundAmount = booking.refundAmount  (из ответа API: cancelBooking, listBookings)
```

- При отмене >12ч до начала: `refundAmount = null` (возврат 100%, отмена бесплатна).
- При отмене ≤12ч до начала: `refundAmount = price × 0.5` (рассчитывает и фиксирует бэкенд).
- Приложение отображает `refundAmount` как справочную информацию: «К возврату: {refundAmount} ₽».
- Фактический возврат — ручной процесс владельца (R11), приложение не инициирует платёж.

### Кнопка-заглушка оплаты

На SCR-003 отображается кнопка «Скоро: онлайн-оплата» (вместо реального платёжного шлюза). Текст из FR-3.3. Кнопка в состоянии `disabled`, ведёт себя как информационная заглушка.

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** слот с `price=3500`, **Когда** открывается SCR-002, **Тогда** отображается «3500 ₽» |
| AC-002 | **Дано** бронь с `refundAmount=1750`, **Когда** открывается SCR-005, **Тогда** в карточке отменённой брони отображается «К возврату: 1750 ₽» |
| AC-003 | **Дано** бронь с `refundAmount=null`, **Когда** открывается SCR-005, **Тогда** сумма возврата не отображается |
| AC-004 | **Дано** открыта SCR-003, **Когда** отображается кнопка оплаты, **Тогда** она показывает «Скоро: онлайн-оплата», disabled |
