## Обзор

Правило расчёта стоимости отмены брони в зависимости от оставшегося до начала класса времени. Определяет, какой текст предупреждения показать клиенту и будет ли удержана часть стоимости. **ВАЖНО: порог — 12 часов, не 2 часа.** Все расчёты выполняются на бэкенде — приложение вычисляет `remaining` для правильного предупреждения, но финальный `refundAmount` получает из ответа API.

### User Story

> Как Клиент, я хочу знать правила возврата при отмене брони,
> чтобы принять осознанное решение об отмене или сохранении брони.

### Бизнес-ценность

- Прозрачные правила отмены (FR-4.1)
- Мотивация отменять заранее (>12ч), освобождая места для других клиентов
- Компенсация студии при поздней отмене (50%)

## Описание логики

### Правило

```
remaining = slot.dateTime − now()

Если remaining > 12 часов:
  → отмена бесплатна
  → refundAmount = null
  → предупреждение: «Отмена бесплатна. Подтвердить отмену?»

Если remaining ≤ 12 часов:
  → возврат 50% стоимости
  → refundAmount = slot.price × 0.5
  → предупреждение: «При отмене возвращается 50% стоимости.
     Сумма к возврату: {refundAmount} ₽.
     Возврат осуществит владелец вручную.
     Подтвердить отмену?»
```

### Ключевое
- Порог — **12 часов** (не 2, не 24).
- `remaining` вычисляется на клиенте для отображения правильного предупреждения.
- Бэкенд при получении `cancelBooking` **пересчитывает** `remaining` и фиксирует `refundAmount` — это защита от «зависшего» на границе 12-часового окна запроса.
- Фактический возврат — ручной процесс владельца (наличные / перевод), не через приложение.

### Отображение в истории

В карточке отменённой брони (SCR-005):
- Если `refundAmount == null` — просто статус «Отменена клиентом».
- Если `refundAmount > 0` — статус + «К возврату: {refundAmount} ₽».

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** до начала класса 24 часа, **Когда** клиент нажимает «Отменить», **Тогда** предупреждение «Отмена бесплатна. Подтвердить отмену?» |
| AC-002 | **Дано** до начала класса 6 часов, **Когда** клиент нажимает «Отменить», **Тогда** предупреждение «При отмене возвращается 50% стоимости. Сумма к возврату: {price/2} ₽.» |
| AC-003 | **Дано** до начала ровно 12 часов 0 минут, **Когда** нажимает «Отменить», **Тогда** предупреждение о возврате 50% (≤12ч) |
| AC-004 | **Дано** до начала 12 часов 1 минута, **Когда** нажимает «Отменить», **Тогда** предупреждение о бесплатной отмене (>12ч) |
| AC-005 | **Дано** запрос отмены «завис» у клиента на границе 12ч, **Когда** бэкенд обрабатывает, **Тогда** refundAmount определяется по фактическому remaining на момент обработки |
