## Обзор
Bottom Sheet для переноса активной брони на другой доступный слот. Открывается из SCR-005 по кнопке «Перенести». Загружает список доступных слотов на 7 дней, исключая текущий слот брони. При выборе нового слота — диалог подтверждения и отправка `transferBooking`. Бэкенд одной транзакцией отменяет старую бронь и создаёт новую. Ответ: `TransferResponse { oldBooking, newBooking }`.

### User Story
> Как Клиент, я хочу переносить бронь на другое занятие в одно действие,
> чтобы быстро корректировать свои планы без ручной цепочки отмен и новых поисков.

### Бизнес-ценность
- Бесшовный перенос — одна операция (FR-4.2)
- Атомарность: старая бронь не теряется при ошибке (ROLLBACK)
- Наследование `equipmentType` — не нужно выбирать повторно

## Используемые запросы

### getBooking
**Тип:** REST
**Метод:** GET
**Спецификация:** [../api/bookings/api.yaml](../api/bookings/api.yaml) → `getBooking`

**Триггер:** Инициализация

**Параметры:**
| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `bookingId` | string (UUID) | Да | Входной параметр | ID брони |

**Обработка ответа:**
| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Успех (200) | — | Получить `slotId` для исключения из списка |
| HTTP 404 | — | Снек «Бронь не найдена», закрыть BS-002 |
| HTTP 401 | — | Редирект на SCR-006 |

### listSlots
**Тип:** REST
**Метод:** GET
**Спецификация:** [../api/slots/api.yaml](../api/slots/api.yaml) → `listSlots`

**Триггер:** Инициализация

**Параметры:**
| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `from` | string (date) | Да | `today()` | Сегодня |
| `to` | string (date) | Да | `today() + 7д` | 7 дней |

**Обработка ответа:**
| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Скелетон-шиммер |
| Успех + items | — | Отфильтровать: исключить `booking.slotId`, `status!="Активен"`, `bookedCount>=capacity`. Применить LOGIC-002 |
| Успех + пусто | — | Empty: «Нет доступных классов для переноса» |
| HTTP 401 | — | Редирект на SCR-006 |
| HTTP 5xx | — | Error state с кнопкой «Обновить» |

### transferBooking
**Тип:** REST
**Метод:** POST
**Спецификация:** [../api/bookings/api.yaml](../api/bookings/api.yaml) → `transferBooking`

**Триггер:** Подтверждение переноса

**Body:**
| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `newSlotId` | string (UUID) | Да | Выбранный слот | ID нового слота |

**Обработка ответа:**
| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Лоадер, блокировка UI |
| Успех (200) | — | Снек «Бронь перенесена», закрыть BS-002, обновить SCR-005 |
| HTTP 409 | `slot_full` | Снек «На выбранном классе нет мест», обновить listSlots |
| HTTP 409 | `booking_not_active` | Снек «Бронь уже отменена или завершена», закрыть |
| HTTP 410 | `slot_cancelled` | Снек «Выбранный класс более недоступен», обновить listSlots |
| HTTP 401 | — | Редирект на SCR-006 |
| HTTP 5xx | — | Снек «Произошла ошибка. Попробуйте позже» |
| Сеть | Нет | Снек «Нет соединения. Проверьте подключение» |

## Элементы экрана

### 1. Текущая бронь
| Элемент | Описание | Источник данных |
|---------|----------|-----------------|
| Меню | Название программы | `booking.slot.menu` (getBooking) |
| Дата и время | Формат: «12 июля, Вс • 14:00» | `booking.slot.dateTime` |

### 2. Карточка слота (для переноса)
| Элемент | Описание | Источник данных |
|---------|----------|-----------------|
| Дата и время | Формат локализованный | `slot.dateTime` (listSlots) |
| Меню | Название программы | `slot.menu` |
| Шеф + рейтинг | «Шеф: {name} ★ {rating}» | `slot.instructor` (inline) |
| Сложность | «Для новичков» / «Для опытных» | `slot.difficulty` |
| Свободные места | «Осталось N из M» | LOGIC-002 |
| Цена | «{price} ₽» | `slot.price` |

**Логика:**
- Доступность: [LOGIC-002](00_Логики/LOGIC-002_Доступность.md) — pre-check. В список только `status="Активен"`, `bookedCount < capacity`, `canBook == true`.
- Фильтрация: [LOGIC-005](00_Логики/LOGIC-005_Фильтрация_по_дате.md) — 7 дней.
- Текущий `booking.slotId` исключается.

### 3. Диалог подтверждения
| Элемент | Действие |
|---------|----------|
| Текст: «Перенести бронь с {старый} на {новый}?» | — |
| Кнопка «Перенести» | → [transferBooking](#transferbooking) |
| Кнопка «Отмена» | Вернуться к списку |

## Критерии приёмки

### Позитивные сценарии
| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** открыта BS-002, **Когда** listSlots загружен, **Тогда** список: исключён текущий слот, исключены `status!="Активен"` и `bookedCount>=capacity` | P0 |
| AC-002 | **Дано** выбран слот, **Когда** подтверждён перенос, **Тогда** transferBooking 200 → снек «Бронь перенесена», старая «Отменена клиентом», новая «Активна» | P0 |
| AC-003 | **Дано** transferBooking 200, **Когда** `TransferResponse { oldBooking, newBooking }`, **Тогда** `equipmentType` новой брони = `equipmentType` старой | P1 |

### Негативные сценарии
| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** transferBooking 409 (slot_full), **Когда** ответ, **Тогда** снек «На выбранном классе нет мест», старая бронь не тронута, обновить список | P0 |
| AC-N02 | **Дано** transferBooking 410 (slot_cancelled), **Когда** ответ, **Тогда** снек «Выбранный класс более недоступен», обновить список | P0 |
| AC-N03 | **Дано** transferBooking 409 (booking_not_active), **Когда** ответ, **Тогда** снек «Бронь уже отменена или завершена», закрыть BS-002 | P1 |
